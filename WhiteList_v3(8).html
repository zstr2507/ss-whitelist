Keamanan Server & Skenario Login

Sistem whitelist ini menggunakan Firebase Authentication + Firestore. Otorisasi berbasis Place Owner (bukan per-player), artinya jika place owner sudah diizinkan, semua player di map itu bisa pakai asset.

✓ Aman
Ganti PC / Browser
Login ulang dengan akun Google yang sama → akses penuh. Firebase Auth berbasis akun Google, bukan device. Sesi otomatis restore jika belum expire.
✓ Aman
Pakai HP (Mobile)
Buka URL dashboard di browser HP → Sign In Google → berfungsi normal. Dashboard sudah responsif untuk mobile. Data Firestore real-time di semua device.
⚠ Perhatian
Multi-Login (Banyak Tab/Browser)
Firebase mendukung multi-session. Jika login di banyak device sekaligus, perubahan di satu device langsung real-time di semua device lainnya via Firestore listener. Tidak ada konflik data.
✓ Aman
Keamanan Roblox Asset
Script server hanya baca Firestore dengan API key yang terbindung ke project spesifik. Orang lain tidak bisa mengakses data whitelist tanpa credentials Firebase. OWNER_UID dikodekan di script, sehingga hanya data owner yang terbaca.
✓ Aman
Otorisasi Place-Level
Pengecekan dilakukan saat server Roblox startup (sekali). PLACE_AUTHORIZED disimpan di memori server. Semua player yang join map tersebut mendapat status yang sama — tidak perlu cek per-player sehingga efisien dan aman.
